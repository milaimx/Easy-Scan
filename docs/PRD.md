# 产品需求文档（PRD）— 防重复条码扫描应用

**版本：V1.0 | 日期：2026-02-28**

## 1. 产品背景与目标

基于现有「专业扫码」App 中的防重复扫描功能，将其独立提炼为一款专注、轻量的条码扫描应用，并新增自定义前缀过滤、本地持久化存储及 WebDAV 自动备份能力。

### 1.1 核心目标
- 防止同一条码被重复扫入，实时提醒操作员
- 支持自定义前缀规则，不合规条码立即报错
- 所有扫描数据本地持久化存储，App 关闭后数据不丢失
- 每次执行「清空数据」前，自动通过 WebDAV 将当前数据备份至服务器
- 提供清晰的计数总览（当前已扫总数始终可见）

## 2. 功能范围（V1.0）

| 功能模块 | 功能点 | 优先级 |
|---------|--------|--------|
| 防重复扫描 | 扫入条码与已有列表比对，重复则报警（声音+震动+红色提示） | P0 |
| 自定义前缀过滤 | 设置一个或多个允许的前缀规则 | P0 |
| 扫描总数显示 | 顶部实时显示已扫合法条码总数 | P0 |
| 本地持久化存储 | 扫描数据写入设备本地，App 重启后数据保留 | P0 |
| 数据列表查看 | 可滚动查看所有已扫条码（序号、条码、扫描时间） | P0 |
| 导出数据 | 将当前数据导出为 TXT 文件到本地 | P1 |
| 清空数据 | 清空前自动触发 WebDAV 备份 | P0 |
| WebDAV 自动备份 | 清空时将数据文件上传至配置好的 WebDAV 服务器 | P0 |
| 设置页 | 文件名模板、条码长度范围、前缀规则、WebDAV 配置 | P0 |

## 3. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 条码重复检测响应时间 < 100ms |
| 存储 | 支持至少 100,000 条记录 |
| 网络 | 离线时可正常扫码，清空时提示无网络 |
| 平台 | Android 8.0+；iOS 13.0+ |
